Isolate scope equal
Controller
Directive
  • Used for Two way binding, generally less likely to be used than @
  • Passes object as attribute from html to directive
  • We are passing object instead of string, any change to the object in directive also changes it in controller and vice versa
  • Notice how changing the text in one textbox changes it in other.
  • One textbox is getting printed in html file and populated with data from controller
  • Second textbox is getting printed in directive and populated with data passed to the directive
  • Here the fruit-name is considered as an attribute of the div, which is passed into the 'xyz' property of the isolate scope.
  • This 'xyz' property is then printed in the template which prints the string value of the 'xyz' property.
  • In the code we have var directive = {scope: {xyz: "=fruitName"}}
  • We get the value of fruit-name attribute from the div, assign it to the 'xyz' property in the isolate scope and then print it.
  • We can also do var directive = {scope: {fruitName: "="}}, where instead of 'xyz' property name, we are using 'fruitName' which matches the attribute fruit-name of the li.
